checker¶

This module connects to a list of websites at regular interval and records connection metrics into a Kafka topic.

This can be run as as aiven-monitor-checker or python -m aiven_monitor.checker :

usage: aiven-monitor-checker [-h] [--config CONFIG]

Connects to a list of websites at regular interval and records connection metrics into a Kafka topic.

optional arguments:
  -h, --help       show this help message and exit
  --config CONFIG  path to the config file. absolute or relative to the working directory. defaults to "checker.ini".

Configuration Options¶

The configuration options are read from a reader.ini file using ConfigParser.

All the following settings must be in a [checker] section of the ini file.

[checker]
#  A name for the location from where the checker is run. Default value is "default".
endpoint = default
# A comma-separated list of Kafka servers. Will connect to "localhost:9092" if not configured.
kafka.bootstrap_servers = "ocalhost:9092
# Kafka topic receiving the produced Measure objects. Default value is "aiven_monitor_measure".
kafka.topic = aiven_monitor_measure
# Path to the PEM certificate of the authority used to authenticate the Kafka servers. Optional.
kafka.ssl.cafile = /path/to/kafka.ca.pem
# Path to the PEM certificate used to authenticate the checker. Optional.
kafka.ssl.certfile = /path/to/kafka.certificate.pem
# Path to the PEM private key used to authenticate the checker. Optional.
kafka.ssl.keyfile = /path/to/kafka.private.pem
# Number of seconds before retrying to connect. Default value is 1.0.
kafka.connect_interval_secs = 1.0

Probe¶

class aiven_monitor.checker.Probe(url: str, interval_secs: float, expected_pattern: Optional[str] = None)¶

The monitoring configuration for a single Url.

url: str¶

The Url this Probe will check.

interval_secs: float¶

The approximate interval between two checks.

This must be strictly positive.

See Schedule for details about the scheduling logic.

expected_pattern: Optional[re.Pattern]¶

An optional regular expression to search in the response.

async check(recorder: aiven_monitor.checker.KafkaRecorder, endpoint: str) → None¶

Runs a new HTTP(s) request on the configured URL and records probing results in a aiven_monitor.Measure.

HTTP/1.1 and HTTP/2 are both supported.

Parameters

  • recorder – A recorder to handle the Measure.

  • endpoint – A name for the place from where this check is running.

Schedule¶

class aiven_monitor.checker.Schedule(probe: aiven_monitor.checker.Probe, base_time: float)¶

Maintains the scheduling state of each Probe inside a Scheduler.

The Scheduler picks the Schedule with the least next_time first.

probe: aiven_monitor.checker.Probe¶

The scheduled Probe.

base_time: float¶

The earliest time the Probe is allowed to be scheduled.

This changes each time advance() is called.

next_time: float¶

The actual time the Probe is scheduled.

This is always between base_time and base_time + Probe.interval_secs.

advance() → None¶

  • Advances base_time by Probe.interval_secs.

  • Updates next_time to stay between base_time and base_time + Probe.interval_secs.

Scheduler¶

class aiven_monitor.checker.Scheduler¶

Elects which Probe object must be run next.

Internally, each added Probe is wrapped in a Schedule.

async add(probes: Iterable[aiven_monitor.checker.Probe]) → None¶

Adds a list of probes.

Each one of them will be scheduled to be run somewhere between trio.current_time() and trio.current_time() + Probe.interval_secs.

async remove(probes: Iterable[aiven_monitor.checker.Probe]) → None¶

Removes a list of probes.

It is not an error to try to remove a probe that was not previously added.

async get_next() → aiven_monitor.checker.Probe¶

Returns the next probe that must be run immediately.

If the earliest available probe needs to be run later, this will internally wait until it’s time to run the probe.

If the scheduling queue changes while waiting, the earliest scheduled probe will be re-elected immediately. This means that you can concurrently add and remove probes and not wait until it’s time to run the previously elected probe.

If the scheduling queue is empty, this will wait until something is added to the queue.

KafkaRecorder¶

class aiven_monitor.checker.KafkaRecorder(bootstrap_servers: str, topic: str, ssl_cafile: Optional[Union[str, pathlib.Path]], ssl_certfile: Optional[Union[str, pathlib.Path]], ssl_keyfile: Optional[Union[str, pathlib.Path]], connect_interval_secs: float = 1.0)¶

A Kafka producer that takes aiven_monitor.Measure objects, and sends them to Kafka as utf-8 json serialized dicts.

Parameters

  • bootstrap_servers – A comma-separated list of Kafka servers.

  • topic – The Kafka topic receiving the aiven_monitor.Measure objects.

  • ssl_cafile – The path to the PEM certificate of the certificate authority used to authenticate the Kafka servers.

  • ssl_certfile – The path to the PEM certificate used to authenticate this producer.

  • ssl_keyfile – The path to the PEM private key used to authenticate this producer.

  • connect_interval_secs – The number of seconds before retrying to connect.

async start() → None¶

Starts the Kafka producer.

If the connection cannot be established, it will endlessly retry while waiting connect_interval_secs between each attempt.

async record(measure: aiven_monitor.Measure) → None¶

Records a aiven_monitor.Measure in Kafka.

This does not wait for the measure to be actually sent.

Because this requires a working Kafka connection, this function will not return until start() has completed.

It is not an error to call record() before or while calling start(). However, this means that if record() is called as a coroutine and not directly awaited, pending measures and coroutines can accumulate in memory until the connection is established.

flush() → None¶

Synchronously waits for all pending events to be sent.
